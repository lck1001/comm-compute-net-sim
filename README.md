# 通信算力网络仿真环境

这是一个轻量化通信算力网络仿真原型，面向 Linux 部署和现场演示。它使用一个 Node.js 进程同时提供仿真后端、数据接口和浏览器展示界面，无需数据库、无需前端构建步骤。

## 已实现能力

- 构建 200 个仿真节点，包含核心算力、边缘节点、接入节点、中继节点。
- 动态组网：节点按区域分布，链路会持续波动，支持链路在线/中断、带宽、时延、丢包率、利用率变化。
- 通信链路模拟与可视化：主画布展示全网拓扑、链路热度、传输流光、任务路径高亮。
- 任务拆解与周边执行：任务从任意在线节点发起，调度层限定 2 跳通联范围，按节点剩余算力/存储与链路质量拆分任务片段。
- 数据接口：提供 REST API 和 SSE 实时推送，展示层通过 `/api/events` 获取状态变化。
- 展示层：包含动态拓扑、节点传输详情、链路详情表、单条任务逐节点传输过程。

## 三层结构

### 数据层

内存保存网络状态，默认包含：

- 200 个节点的类型、位置、区域、状态、总算力、剩余算力、总存储、剩余存储、收发速率。
- 节点间通信链路状态：端点、带宽、时延、丢包率、负载、在线/中断。
- 最近任务队列和每个任务分片的执行进度。

### 调度层

调度入口位于 `POST /api/tasks`，也会周期性自动注入任务。算法步骤：

1. 以任务发起节点为中心寻找 2 跳内通联节点。
2. 过滤离线节点、链路不可达节点、资源过低节点。
3. 对候选节点评分：剩余算力、剩余存储、路径瓶颈带宽、路径时延、丢包率、链路利用率。
4. 选择前若干候选节点，将任务拆分为多个分片。
5. 更新分片节点资源占用，并在任务完成后释放资源。

### 展示层

`public/` 下是无依赖浏览器界面：

- `动态拓扑`：Canvas 绘制 200 节点拓扑、链路状态和任务路径。
- `链路详情`：表格展示单条链路实时指标。
- `逐节点传输`：展示当前任务的分片路径、进度和节点传递过程。

## Linux 部署

要求 Node.js 20 或更高版本。

```bash
cd comm-compute-net-sim
npm start
```

默认监听：

```text
http://127.0.0.1:4173
```

如需指定端口：

```bash
PORT=8080 npm start
```

## API

### 获取全网状态

```http
GET /api/state
```

### 实时状态推送

```http
GET /api/events
```

返回 Server-Sent Events，事件类型包括：

- `state`：全网快照。
- `task`：新任务产生。

### 注入任务

```http
POST /api/tasks
Content-Type: application/json

{
  "origin": "N001",
  "compute": 960,
  "storage": 420,
  "data": 520,
  "priority": "高"
}
```

返回 `201` 表示调度接受，`409` 表示 2 跳范围内资源或链路质量不足。

## 演示建议

1. 打开首页后先看动态拓扑，观察链路流光和节点传输脉冲。
2. 在左侧选择发起节点，调大算力/存储/数据量后点击“注入任务”。
3. 切到“逐节点传输”，讲解任务如何被拆成分片并在 2 跳节点内执行。
4. 切到“链路详情”，展示链路时延、丢包率和负载会动态变化。
